# Part 1: Introduction to Object-Oriented Programming (OOP)

Object-Oriented Programming (OOP) is a programming paradigm that organizes software design around data, or objects, rather than functions and logic (which is followed by Procedural Programming).

An object contains:
- **Data (State)**
- **Behavior**

---

## What is an Object?

An object is a real-world entity that has a state and behavior.

### Example: Car Object
- **Data (State)**: Color, Brand, Model, Year
- **Behavior**: Start, Stop, Accelerate, Brake

---

## Features of OOP

There are four main pillars of Object-Oriented Programming:

1. **Encapsulation**: Wrapping data (variables) and methods (behavior) together into a single unit (class).
2. **Abstraction**: Hiding internal details and showing only the essential features to the user.
3. **Inheritance**: The mechanism by which one class acquires the properties and behaviors of another class.
4. **Polymorphism**: The ability of a message or action to be processed in more than one way (e.g., method overloading and method overriding).

---

## Characteristics of OOP

1. **Object-based**
2. **Class-based**
3. **Supports inheritance**
4. **Supports polymorphism**
5. **Supports abstraction**
6. **Supports encapsulation**
7. **Reusable code**
8. **Easy maintenance**
9. **Modular design**
10. **Better security**

---

## Advantages of OOP

- **Code Reusability**: Classes and components can be reused across programs.
- **Better Security**: Data hiding via encapsulation keeps data safe from unauthorized access.
- **Easy Maintenance**: Easy to manage and update due to modularity.
- **Scalability**: Highly scalable from small to large-scale applications.
- **Modularity**: Code is broken down into independent modules/objects.

---

## Disadvantages of OOP

- **More Memory Consumption**: Objects require memory allocation.
- **Slower Execution**: Typically slower than procedural programming for very small, simple programs due to overhead.
