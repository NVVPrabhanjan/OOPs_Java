Polymorphism is the third pillar of Object-Oriented Programming (OOP).

The word Polymorphism comes from two Greek words:
Poly = Many
Morph = Forms

So,
Polymorphism means "one thing having many forms."

In Java, the same method name can behave differently depending on the situation.

1. Compile Time Polymorphism: Method Overloading 
2. Runtime Polymorphism: Method Overriding

Compile Time Polymorphism

Rules of Method Overloading

Methods must differ in:

Number of parameters
Type of parameters
Order of parameters

Can we overload main() method?
Yes!

Runtime Polymorphism

Rules:
1. Same method name
2. Same parameters
3. Same return type (or covariant return type)
4. Same inheritance hierarchy

Cannot override

1. Final methods
2. Static methods (they are hidden, not overridden)
3. Private methods

Runtime Polymorphism is also called as
→ Dynamic Binding
→ Late Binding
→ Method Overriding


Upcasting
Converting child reference into parent reference.

Dog d = new Dog();
Animal a = d;

This is called Upcasting

Why Upcasting?
Allows one reference to point to different child objects.
Animal animals[] = {

    new Dog(),

    new Cat(),

    new Cow()
};


for(Animal a : animals)
      a.sound();

Output :
Bark
Meow
Moo

Very Powerful


Downcasting
Converting parent reference into child reference.

Animal a = new Dog();
Dog d = (Dog)a;

Explicit cast required.

The below example is Wrong

Animal a = new Animal();
Dog d = (Dog)a;

Runtime Error

ClassCastException

Because the actual object is not a Dog.
